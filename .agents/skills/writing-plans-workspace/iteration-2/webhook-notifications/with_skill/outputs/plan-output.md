# Webhook Notifications on Phase Change — Implementation Plan

**Spec:** [WEBHOOK_NOTIFICATIONS_SPEC](../../specs/WEBHOOK_NOTIFICATIONS_SPEC.md)
**Goal:** Enable external systems to receive HTTP POST callbacks when a workshop phase changes, configured per-workshop by facilitators.
**Architecture:** Add `webhook_url` and `webhook_enabled` columns to the `workshops` table. A new `WebhookService` handles async HTTP dispatch via `httpx`. Each `advance-to-*` endpoint in the workshops router uses FastAPI `BackgroundTasks` to fire the webhook after a successful phase commit, ensuring non-blocking delivery that never rolls back the phase change.
**Success Criteria Targeted:**
- SC-CFG-1: Facilitator can set a webhook URL for a workshop via PUT endpoint
- SC-CFG-2: Facilitator can retrieve current webhook config via GET endpoint
- SC-CFG-3: Facilitator can delete webhook config via DELETE endpoint
- SC-CFG-4: Webhook URL is validated (must be valid HTTP/HTTPS URL)
- SC-CFG-5: Non-facilitator roles receive 403 when configuring webhooks
- SC-CFG-6: Webhook can be enabled/disabled without removing the URL
- SC-DLV-1: Phase change triggers HTTP POST to configured webhook URL
- SC-DLV-2: Webhook payload includes event type, workshop_id, previous_phase, new_phase, timestamp, workshop_name
- SC-DLV-3: Webhook delivery is asynchronous (phase advance response not delayed)
- SC-DLV-4: Failed webhook delivery does not roll back phase change
- SC-DLV-5: Webhook delivery failure is logged at WARNING level
- SC-DLV-6: Webhook request times out after 5 seconds
- SC-DLV-7: No webhook sent when URL is not configured
- SC-DLV-8: No webhook sent when webhook is disabled
- SC-EDGE-1: All six phase transitions trigger webhooks (intake->discovery, discovery->rubric, rubric->annotation, annotation->results, annotation->judge_tuning, judge_tuning->unity_volume)
- SC-EDGE-2: Generic advance_workshop_phase endpoint triggers exactly one webhook (not duplicated)
- SC-EDGE-3: Concurrent phase advances do not cause duplicate webhooks

def get_webhook_config(self, workshop_id: str) -> Optional[dict]:
    """Get webhook configuration for a workshop."""
    db_workshop = self.db.query(WorkshopDB).filter(WorkshopDB.id == workshop_id).first()
    if not db_workshop:
        return None
    return {
        "url": db_workshop.webhook_url,
        "enabled": db_workshop.webhook_enabled or False,
    }

def set_webhook_config(self, workshop_id: str, url: str, enabled: bool = True) -> Optional[dict]:
    """Set webhook configuration for a workshop."""
    db_workshop = self.db.query(WorkshopDB).filter(WorkshopDB.id == workshop_id).first()
    if not db_workshop:
        return None
    db_workshop.webhook_url = url
    db_workshop.webhook_enabled = enabled
    self.db.commit()
    self.db.refresh(db_workshop)
    return {
        "url": db_workshop.webhook_url,
        "enabled": db_workshop.webhook_enabled,
    }

def delete_webhook_config(self, workshop_id: str) -> bool:
    """Delete webhook configuration for a workshop."""
    db_workshop = self.db.query(WorkshopDB).filter(WorkshopDB.id == workshop_id).first()
    if not db_workshop:
        return False
    db_workshop.webhook_url = None
    db_workshop.webhook_enabled = False
    self.db.commit()
    return True
```

"""Service for sending webhook notifications on workshop events."""

import logging
from datetime import datetime, timezone

import httpx

from server.models import WorkshopPhase

logger = logging.getLogger(__name__)

WEBHOOK_TIMEOUT_SECONDS = 5.0


class WebhookService:
    """Handles outbound webhook HTTP POST notifications.

    All methods are fire-and-forget: they never raise exceptions
    and never block the caller beyond the timeout.
    """

    async def notify_phase_change(
        self,
        workshop_id: str,
        workshop_name: str,
        webhook_url: str,
        previous_phase: WorkshopPhase,
        new_phase: WorkshopPhase,
    ) -> None:
        """Send a phase_changed webhook notification.

        Args:
            workshop_id: The workshop that changed phase.
            workshop_name: Human-readable workshop name.
            webhook_url: Target URL for the POST request.
            previous_phase: Phase before the transition.
            new_phase: Phase after the transition.

        Returns:
            None. Never raises — all errors are logged at WARNING level.
        """
        payload = {
            "event": "phase_changed",
            "workshop_id": workshop_id,
            "previous_phase": str(previous_phase),
            "new_phase": str(new_phase),
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "workshop_name": workshop_name,
        }

        try:
            async with httpx.AsyncClient() as client:
                response = await client.post(
                    webhook_url,
                    json=payload,
                    timeout=WEBHOOK_TIMEOUT_SECONDS,
                )
                response.raise_for_status()
                logger.info(
                    "Webhook delivered successfully for workshop %s: %s -> %s",
                    workshop_id,
                    previous_phase,
                    new_phase,
                )
        except Exception:
            logger.warning(
                "Webhook delivery failed for workshop %s to %s: %s -> %s",
                workshop_id,
                webhook_url,
                previous_phase,
                new_phase,
                exc_info=True,
            )
```

Then modify each of the six `advance-to-*` endpoints to accept `BackgroundTasks` and dispatch the webhook after a successful phase update. The pattern is the same for all six. Example for `advance_to_discovery`:

```python
@router.post("/{workshop_id}/advance-to-discovery")
async def advance_to_discovery(
    workshop_id: str,
    db: Session = Depends(get_db),
    background_tasks: BackgroundTasks = BackgroundTasks(),
):
    """Advance workshop from INTAKE to DISCOVERY phase (facilitator only)."""
    db_service = DatabaseService(db)
    workshop = db_service.get_workshop(workshop_id)
    if not workshop:
        raise HTTPException(status_code=404, detail="Workshop not found")

    previous_phase = workshop.current_phase

    # Validation: Check prerequisites
    if workshop.current_phase != WorkshopPhase.INTAKE:
        raise HTTPException(status_code=400, detail=f"Cannot advance to discovery from {workshop.current_phase} phase")

    # Check if traces exist
    traces = db_service.get_traces(workshop_id)
    if len(traces) == 0:
        raise HTTPException(status_code=400, detail="Cannot start discovery phase: No traces uploaded to workshop")

    # Update workshop phase
    db_service.update_workshop_phase(workshop_id, WorkshopPhase.DISCOVERY)

    # Fire webhook if configured
    webhook_config = db_service.get_webhook_config(workshop_id)
    if webhook_config and webhook_config["url"] and webhook_config["enabled"]:
        background_tasks.add_task(
            webhook_service.notify_phase_change,
            workshop_id=workshop_id,
            workshop_name=workshop.name,
            webhook_url=webhook_config["url"],
            previous_phase=previous_phase,
            new_phase=WorkshopPhase.DISCOVERY,
        )

    return {
        "message": "Workshop advanced to discovery phase",
        "phase": "discovery",
        "workshop_id": workshop_id,
        "traces_available": len(traces),
    }
```

Apply the same pattern to all six endpoints:
1. `advance_to_discovery` — intake -> discovery
2. `advance_to_rubric` — discovery -> rubric
3. `advance_to_annotation` — rubric -> annotation
4. `advance_to_results` — annotation -> results
5. `advance_to_judge_tuning` — annotation/results -> judge_tuning
6. `advance_to_unity_volume` — judge_tuning -> unity_volume

**Important:** The generic `advance_workshop_phase` endpoint delegates to these specific endpoints, so it does NOT need its own webhook dispatch — the webhook fires inside the delegated call, preventing duplicates (SC-EDGE-2).

For `advance_to_judge_tuning` and `advance_to_unity_volume`, skip the webhook dispatch if `already_in_phase` is True (idempotent case — no actual transition occurred).
